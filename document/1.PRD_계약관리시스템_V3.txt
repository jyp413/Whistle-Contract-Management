# 주차단속 계약관리 시스템 PRD (개정판 v3.0)

> **개정 이력**
> - v1.0: 초안 (기능 요구사항 정의)
> - v2.0: 회계·내부통제 관점 검토 의견 반영
> - **v3.0: 사용자 피드백 반영 (현재 버전)**
>   - KPI를 3가지(계약완료/체결중/갱신중)로 단순화 및 drill-down UI 도입
>   - 계약기간 연장 기능(Extend) 신규 추가
>   - 상태 진입 조건 정밀화: '체결중'은 계약사항 등록만(파일 미업로드), '계약완료'는 파일 업로드 + 확인 팝업
>   - 오작업 수정(Correction) 기능 추가
>   - 감사 대응 시나리오 제거

---

## 1. 제품 개요

전국 지방자치단체(시/군/구)와 체결되는 **주차단속 관련 계약**을 통합 관리하는 웹 기반 계약관리 시스템입니다. 현재 엑셀과 개별 파일로 분산 관리되는 계약서·첨부서류·갱신 이력을 단일 플랫폼에 집중시키고, 대시보드를 통한 실시간 가시성과 권한 기반 접근 통제를 제공합니다.

### 1.1 핵심 가치

| 핵심 가치 | 내용 |
|---|---|
| 중앙화 (Centralization) | 지자체별 계약 자료의 단일 진실 공급원(SSOT) 확보 |
| 가시성 (Visibility) | 계약 상태와 진행률에 대한 실시간 대시보드 제공 |
| 통제 (Control) | 역할 기반 권한 통제 및 상태 변경 규칙 강제 |
| 추적성 (Traceability) | 변경 이력의 완전한 추적 보장 |

---

## 2. 문제 정의 (Pain Points)

- 계약서와 첨부파일이 분산되어 추적이 어려움
- 버전 이력 관리를 위한 중앙 시스템 부재
- 계약 진행 상황 및 상태 업데이트의 실시간 추적 불가
- 엑셀 기반 수기 관리에 따른 비효율 및 휴먼 에러 위험

---

## 3. 목표 및 핵심 KPI

### 3.1 핵심 KPI (대시보드 표시 지표)

전체 지자체 수를 기준으로 다음 3가지 지표를 표시합니다.

| KPI | 정의 | 산식 |
|---|---|---|
| 계약완료 계약 수 | 계약완료 상태인 계약 건수 | COUNT(상태 = 계약완료) |
| 체결중 계약 수 | 체결중 상태인 계약 건수 | COUNT(상태 = 체결중) |
| 갱신중 계약 수 | 갱신중 상태인 계약 건수 | COUNT(상태 = 갱신중) |

**Drill-down UI 요구사항**

- 각 KPI 카드는 클릭 가능한 영역으로 구현
- 클릭 시 해당 상태에 속하는 **지자체 리스트(Drill-down List)** 가 모달 또는 별도 화면으로 노출
- 리스트에는 다음 컬럼 표시: 지자체명(시/군/구), 계약 건 ID, 계약체결일, 계약시작일, 계약만료일, 최종 수정일시
- 리스트 항목 클릭 시 해당 계약의 상세 페이지로 이동
- 리스트 자체에서 정렬·검색·엑셀 내보내기 가능

### 3.2 운영 목표

| 구분 | 목표 | 측정 지표 |
|---|---|---|
| 중앙화 | 지자체 계약 통합 저장소 구축 | 시스템 등록 지자체 수 / 관리 대상 전체 지자체 수 |
| 효율성 | 수기 관리 업무 감축 | (도입 전 평균 관리시간 − 도입 후 평균 관리시간) ÷ 도입 전 평균 관리시간 |
| 정확성 | 상태 변경 규칙 준수 | 비정상 상태 전이 발생 건수 (목표: 0) |
| 적시성 | 만료 도래 계약 적시 대응 | 만료일 경과 후에도 후속 조치(연장/갱신/종료) 미착수 건수 |

## 4. 데이터 모델 핵심 정의

### 4.1 계약(Contract) 단위 정의

상태 관리의 단위는 **지자체가 아닌 "계약 건(Contract)"** 입니다.

- 하나의 지자체는 여러 개의 계약 건을 보유할 수 있음
- 각 계약 건은 독립된 상태(계약완료/체결중/갱신중/종료)를 가짐
- 갱신 협상 중에도 기존 계약완료 건은 유효하게 공존 가능

### 4.2 계약일자 정의

| 항목 | 정의 | 필수 여부 |
|---|---|---|
| 계약체결일 (Signed Date) | 양 당사자가 계약서에 서명한 날짜 | 필수 |
| 계약시작일 (Effective Date) | 계약 효력이 발생하는 시점 | 필수 |
| 계약만료일 (Expiry Date) | 계약 효력이 종료되는 시점 | 필수 |
| **연장 후 만료일 (Extended Expiry Date)** | **'계약기간 연장'이 실행된 경우의 새 만료일** | **선택 (연장 시 자동 기록)** |
| 업로드일 (Uploaded At) | 시스템에 파일이 등록된 시점 | 자동 기록 |

### 4.3 계약 상태 정의 및 진입 조건

| 상태 | 의미 | **진입 조건** |
|---|---|---|
| 체결중 (In Progress) | 신규 계약 체결을 진행 중 | **담당자가 계약사항(지자체, 계약일자 등 메타데이터)을 등록한 상태. 파일은 업로드되지 않은 상태.** |
| 계약완료 (Completed) | 계약 체결이 완료되어 효력이 유효한 상태 | **PDF 파일 업로드 후, 시스템이 띄우는 "계약완료로 상태를 변경하시겠습니까?" 확인 팝업에서 사용자가 확인을 선택하여 상태 변경이 등록된 시점.** |
| 갱신중 (Updating) | 기존 계약완료 건의 갱신을 진행 중 | 직전 상태가 반드시 '계약완료'인 계약 건에 대해 갱신 착수 시. 신규 계약 건이 별도로 생성되며 초기 상태는 '갱신중'. |
| 종료 (Terminated) | 계약이 만료, 해지, 또는 폐기된 상태 | 계약완료의 만료일 도래(연장·갱신 미실행 시), 체결중·갱신중에서 폐기 처리 시 |

### 4.4 상태 전이도 (State Transition Diagram)

```
       [계약사항 등록 (파일 미업로드)]
                    │
                    ▼
              ┌──────────┐
              │   체결중  │──────────┐
              └─────┬────┘          │
        파일업로드  │               │ 체결 폐기
        + 확인팝업 │               ▼
                    ▼          ┌──────┐
              ┌──────────┐     │ 종료 │◀────┐
              │  계약완료 │────▶└──────┘     │
              └─────┬────┘ 만료(연장·갱신 X)  │
                    │                         │
        ┌───────────┼───────────┐             │
        │           │           │             │
   [기간연장]   [갱신착수]   [만료]           │
        │           │                         │
        ▼           ▼                         │
   계약완료     ┌──────────┐  갱신폐기        │
   (만료일      │   갱신중  │──────────────────┘
    연장 갱신)  └─────┬────┘
                      │ 갱신 성공
                      ▼
                ┌──────────┐
                │  계약완료 │ (신규 계약 건 생성)
                └──────────┘
```

### 4.5 전이 규칙 표

| From → To | 허용 여부 | 트리거 | 비고 |
|---|---|---|---|
| (없음) → 체결중 | ✅ | 담당자가 계약사항 등록 | 파일 미업로드 상태 |
| 체결중 → 계약완료 | ✅ | PDF 업로드 + 확인 팝업 승인 | 시스템이 자동 팝업 노출 |
| 체결중 → 종료 | ✅ | 담당자 폐기 처리 | 사유 입력 권장 |
| **계약완료 → 계약완료 (기간 연장)** | ✅ | **담당자가 '계약기간 연장' 버튼 클릭** | **만료일이 갱신되며, 동일 계약 건 유지. 연장 이력은 활동 로그에 기록.** |
| 계약완료 → 갱신중 | ✅ | 담당자가 '갱신 착수' 실행 | 신규 계약 건이 갱신중 상태로 생성, 기존 건은 만료일까지 계약완료 유지 |
| 계약완료 → 종료 | ✅ | 만료일 도래 시 자동 또는 수동 해지 | 연장·갱신 미실행 시 |
| 갱신중 → 계약완료 | ✅ | 갱신 협상 완료, 파일 업로드 + 확인 팝업 승인 | 기존 건은 만료일에 자동 종료 |
| 갱신중 → 종료 | ✅ | 갱신 폐기 처리 | |
| 그 외 모든 전이 | ❌ | — | 시스템 거부 |

### 4.6 오작업 수정(Correction) 기능

상태 변경이 잘못 등록된 경우를 대비한 보정 기능을 제공합니다.

| 항목 | 내용 |
|---|---|
| 권한 | Master, Accounting |
| 적용 범위 | 직전 상태 변경에 한해 되돌리기 가능 (Undo). 그 이전 이력은 Master만 보정 가능. |
| UI | 계약 상세 화면 내 '상태 보정' 버튼. 클릭 시 사유 입력 필수. |
| 처리 | 상태를 직전 값으로 복원하되, 원본 변경 이력은 활동 로그에 그대로 보존하고 보정 이벤트를 추가 기록 (이력 무결성 보장) |
| 제약 | '계약완료 → 갱신중' 전이로 신규 계약 건이 생성된 경우, 신규 건이 미사용 상태일 때만 보정 가능 |

## 5. 사용자 페르소나 및 시나리오

### 5.1 페르소나

| 역할 | 설명 |
|---|---|
| Master (관리자) | 시스템 전반을 운영하는 최상위 관리자. 사용자 관리 권한 보유. |
| Accounting (회계 담당자) | 계약 등록·수정·갱신·연장을 수행하는 실무자. |
| Viewer (조회자) | 계약 현황을 열람만 하는 사용자. 다운로드 불가. |

### 5.2 주요 시나리오

**시나리오 1 — 신규 계약 등록 (Accounting)**
회계 담당자가 'A시'와 신규 계약을 협의 중인 단계에서, 시스템에 지자체와 계약일자(체결일·시작일·만료일)를 입력하여 계약사항을 등록합니다. 이 시점에서 계약은 자동으로 **'체결중'** 상태가 되며, 대시보드의 '체결중' KPI에 반영됩니다.

**시나리오 2 — 계약 완료 처리 (Accounting)**
협의가 마무리되어 서명된 계약서 PDF를 해당 계약 건에 업로드하면, 시스템이 **"계약완료로 상태를 변경하시겠습니까?"** 확인 팝업을 자동으로 띄웁니다. 담당자가 '확인'을 선택하면 상태가 **'계약완료'** 로 변경되어 등록되며, 대시보드 KPI가 자동 갱신됩니다.

**시나리오 3 — 계약기간 연장 (Accounting)**
B군의 기존 계약(계약완료, 만료일 2026-06-30)이 만료에 임박했고 갱신이 아닌 단순 기간 연장으로 합의된 경우, 담당자는 계약 상세 화면의 **'계약기간 연장'** 버튼을 클릭합니다. 새 만료일을 입력하면 동일 계약 건의 만료일이 갱신되며, 상태는 '계약완료'를 유지합니다. 연장 이력은 활동 로그에 기록됩니다.

**시나리오 4 — 계약 갱신 (Accounting)**
C구의 기존 계약 만료가 도래하여 신규 조건으로 갱신하는 경우, 담당자는 '갱신 착수'를 실행합니다. 신규 계약 건이 **'갱신중'** 상태로 별도 생성되고, 기존 건은 만료일까지 '계약완료'로 유지됩니다. 갱신 협상이 완료되어 신규 계약서를 업로드하면 확인 팝업을 거쳐 '계약완료'로 전환됩니다.

**시나리오 5 — 오작업 보정 (Accounting)**
담당자가 실수로 다른 지자체의 상태를 '계약완료'로 변경한 경우, 해당 계약 상세 화면에서 **'상태 보정'** 버튼을 누르고 사유를 입력하여 직전 상태로 되돌립니다. 원본 변경 이력과 보정 이벤트 모두 활동 로그에 보존됩니다.

**시나리오 6 — KPI Drill-down 조회 (전 권한)**
사용자가 대시보드의 '체결중 12건' KPI 카드를 클릭하면 12개 지자체 리스트가 모달로 표시됩니다. 리스트에서 특정 지자체를 클릭하면 해당 계약의 상세 화면으로 이동합니다.

**시나리오 7 — 월별 보고서 작성 (Master)**
관리자가 대시보드에서 기간 필터를 적용한 후, 엑셀로 내보내기를 실행하여 보고용 자료를 생성합니다.

**시나리오 8 — 현황 조회 (Viewer)**
조회 권한 사용자가 대시보드와 계약 현황을 확인합니다. 다운로드 버튼은 비활성 상태로 노출됩니다.

## 7. 핵심 기능 명세 (Must-Have)

### 7.1 인증 및 권한 관리

- Supabase Auth 기반 이메일/비밀번호 로그인
- 역할 기반 접근 통제(RBAC): Master / Accounting / Viewer
- 세션 만료 시 자동 로그아웃
- 비밀번호 재설정 (이메일 링크)

### 7.2 지자체 기반 계약 관리

- 시/군/구 계층 분류 (**'구' 우선 분류 규칙** 적용)
- 지자체별 독립 계약 보드
- 한 지자체 내 다수의 계약 건 보유 가능
- 계약 건별 메타데이터: 계약체결일, 시작일, 만료일, 연장 후 만료일, 상태, 비고

### 7.3 대시보드 (핵심 기능)

#### 7.3.1 KPI 카드 (3종)

대시보드 상단에 다음 3개의 클릭 가능한 KPI 카드를 표시합니다.

| KPI 카드 | 표시 정보 | 시각 강조 |
|---|---|---|
| 계약완료 | 계약완료 상태 계약 수 / 전체 계약 수 | 녹색 |
| 체결중 | 체결중 상태 계약 수 / 전체 계약 수 | 주황색 |
| 갱신중 | 갱신중 상태 계약 수 / 전체 계약 수 | 파란색 |

> 분모는 **전체 등록 계약 수**(종료 제외)를 기준으로 합니다.

#### 7.3.2 KPI Drill-down UI

- 각 KPI 카드 클릭 → 해당 상태의 지자체/계약 리스트가 모달 형태로 노출
- 리스트 컬럼: 지자체명, 계약 건 ID, 계약체결일, 계약시작일, 계약만료일, 최종 수정일시
- 리스트 기능:
  - 컬럼별 정렬
  - 지자체명 키워드 검색
  - 엑셀 내보내기 (Master, Accounting)
  - 항목 클릭 시 계약 상세 페이지로 이동

#### 7.3.3 보조 정보 영역

KPI 외에 다음 정보를 대시보드에 함께 표시합니다.

| 영역 | 내용 |
|---|---|
| 만료 임박 계약 | 계약완료 상태 중 만료일이 60일 이내인 계약 리스트 (연장·갱신 의사결정 지원) |
| 최근 활동 | 최근 7일 내의 상태 변경, 파일 업로드, 연장 이력 |
| 필터 | 지자체, 상태, 계약체결일 범위, 만료일 범위 |

### 7.4 계약 상태 관리 (인터랙티브)

#### 7.4.1 상태 변경 트리거

| 전이 | 트리거 방식 |
|---|---|
| (없음) → 체결중 | 신규 계약 등록 폼 제출 시 자동 진입 |
| 체결중 → 계약완료 | PDF 업로드 → 시스템이 자동으로 확인 팝업 노출 → 사용자 '확인' → 상태 변경 |
| 계약완료 → 계약완료 (연장) | 계약 상세 화면의 '계약기간 연장' 버튼 → 새 만료일 입력 |
| 계약완료 → 갱신중 | '갱신 착수' 버튼 → 신규 계약 건 자동 생성 |
| 모든 종료 전이 | 폐기/해지 버튼 → 사유 입력 |

#### 7.4.2 확인 팝업 (계약완료 전환)

- 트리거: 체결중 또는 갱신중 상태에서 PDF 업로드 완료 시
- 팝업 메시지: "계약 파일이 업로드되었습니다. 계약 상태를 '계약완료'로 변경하시겠습니까?"
- 옵션: [확인] / [나중에]
- '확인' 선택 시: 상태 변경 + 활동 로그 기록
- '나중에' 선택 시: 파일은 업로드된 상태로 유지, 상태는 변경되지 않음. 사용자가 추후 수동으로 상태 변경 가능

#### 7.4.3 상태 변경 규칙 강제

- 4.5절 전이 규칙 표에 정의된 전이만 시스템이 허용
- 비허용 전이 시도 시 명확한 오류 메시지 반환
- 모든 상태 변경은 활동 로그에 자동 기록

### 7.5 계약기간 연장 (Extend) — 신규 기능

| 항목 | 내용 |
|---|---|
| 적용 대상 | 상태가 '계약완료'인 계약 건 |
| UI 위치 | 계약 상세 화면 내 '계약기간 연장' 버튼 |
| 입력 항목 | 새 만료일(필수), 연장 사유(권장) |
| 처리 | 동일 계약 건의 만료일을 새 값으로 갱신, 상태는 '계약완료' 유지 |
| 권한 | Master, Accounting |
| 이력 관리 | 연장 전 만료일, 연장 후 만료일, 연장자, 연장일시, 사유를 활동 로그에 기록 |
| 제약 | 새 만료일은 기존 만료일보다 미래 시점이어야 함 |
| 다회 연장 | 허용 (복수 연장 시 모든 이력이 보존됨) |

### 7.6 상태 보정 (Correction) — 신규 기능

| 항목 | 내용 |
|---|---|
| UI 위치 | 계약 상세 화면 내 '상태 보정' 버튼 |
| 권한 | Accounting (직전 상태 변경에 한해), Master (전체 이력 보정 가능) |
| 입력 항목 | 보정 사유(필수) |
| 처리 | 상태를 직전 값으로 복원하되 원본 변경 이력은 그대로 보존, 보정 이벤트를 활동 로그에 추가 |
| 제약 | '계약완료 → 갱신중' 전이로 신규 계약 건이 생성된 경우, 신규 건이 미사용 상태일 때만 보정 가능 |

### 7.7 파일 관리

- **PDF 전용** 업로드 (확장자 및 MIME 타입 검증)
- 단일 파일 최대 50MB
- 동일 계약 건에 대한 다수 버전 업로드 지원 (모든 버전 보존)
- 버전 식별: 업로드일시 + 업로더
- PDF 인라인 미리보기 지원
- 권한 기반 다운로드 통제

### 7.8 일괄 다운로드 (ZIP)

- 선택된 지자체의 계약 파일을 ZIP으로 패키징
- 다운로드 범위 옵션:
  - 옵션 A: 모든 계약 건의 **최신 버전만** (기본값)
  - 옵션 B: 모든 계약 건의 **모든 버전**
  - 옵션 C: **특정 상태**의 계약 건만 (예: 계약완료만)
- ZIP 내부 폴더 구조: `/{지자체명}/{계약건ID}_{체결일}/{파일명}`
- Master, Accounting만 실행 가능
- 대용량 처리: 비동기 작업 + 완료 후 다운로드 링크 제공

### 7.9 데이터 내보내기 (엑셀)

- 대시보드 또는 KPI Drill-down 리스트에 적용된 필터 기준으로 내보내기
- 포함 컬럼: 지자체명, 계약 건 ID, 상태, 계약체결일, 시작일, 만료일, 연장 후 만료일, 최종 수정일시
- Master, Accounting만 실행 가능

### 7.10 활동 로그

- 기록 대상 이벤트:
  - 로그인/로그아웃
  - 계약 등록·수정·삭제
  - 상태 변경 (이전 상태, 신규 상태 포함)
  - **계약기간 연장 (연장 전·후 만료일, 사유 포함)**
  - **상태 보정 (보정 전·후 상태, 사유 포함)**
  - 파일 업로드·다운로드 (개별 및 ZIP)
  - 권한 변경
- 기록 항목: 이벤트 ID, 발생일시, 사용자, 대상 객체, 이벤트 유형, 이전 값, 변경 후 값, IP 주소
- Master는 전체 조회, Accounting은 본인 활동만 조회
- 로그는 **수정·삭제 불가** (불변성 보장)
- 엑셀 내보내기 지원

### 7.11 동시성 처리

- 동일 계약 건에 대한 동시 수정 방지: **낙관적 락(Optimistic Lock)** 적용
- 각 계약 레코드에 버전 번호(또는 updated_at) 부여
- 수정 시점에 버전 번호 검증, 불일치 시 충돌 메시지 반환

### 7.12 데이터 보존 정책

- 계약 데이터: 만료일 기준 최소 10년 보존
- '종료' 상태 계약은 보존 기간 내에는 소프트 삭제만 허용
- 보존 기간 경과 후 영구 삭제 가능 (Master 권한)
- 활동 로그: 최소 5년 보존
